# SOP: 자연어 기반 구글 캘린더 일정 관리 시스템 지침서 (directives/chatbot_sop.md)

본 지침서는 사용자의 자연어 입력(채팅 메시지)을 분석하여 구글 캘린더 API가 안전하고 결정론적으로 실행할 수 있도록 정형화된 JSON 규격으로 변환하는 절차(Standard Operating Procedure)를 규정합니다.

---

## 🎯 최종 목표 (Objective)
- 사용자가 던지는 자연어(예: "내일 아침 10시에 회의 등록해줘", "이번 주 일정 다 보여줘", "오후 3시 미팅 취소해줘")를 해석하여, 어떤 **행동(Action)**을 수행해야 하는지 분류하고 관련 **인자(Arguments)**를 정형화된 데이터로 정확히 추출합니다.

---

## 📥 입력값 및 요구 정보 (Inputs)
- `user_message`: 사용자가 채팅창에 입력한 대화 텍스트 원문.
- `current_time`: 상대적인 시간 부사("오늘", "내일", "다음 주 월요일" 등)를 절대 날짜 및 시간으로 계산하기 위한 기준 일시 (ISO 8601 포맷: `YYYY-MM-DDTHH:MM:SS+09:00`).

---

## ⚙️ 의도(Intent) 분류 및 JSON 규격 설계

사용자의 자연어 메시지는 반드시 아래 4가지 행동 중 하나로 해석되어야 하며, 정의된 JSON 규격을 완벽히 준수해야 합니다.

### 1. 일정 등록 (Action: `ADD`)
- **조건**: 사용자가 새로운 일정을 생성하고자 할 때.
- **필수 인자**:
  - `summary`: 일정 제목 (문자열)
  - `start_time`: 일정 시작 시간 (ISO 8601 포맷: `YYYY-MM-DDTHH:MM:SS`)
  - `end_time`: 일정 종료 시간 (ISO 8601 포맷, 명시되지 않은 경우 시작 시간 기준 1시간 뒤로 자동 계산)
  - `description`: 일정 설명 (선택 사항)

```json
{
  "action": "ADD",
  "arguments": {
    "summary": "마케팅 전략 회의",
    "start_time": "2026-05-27T10:00:00",
    "end_time": "2026-05-27T11:00:00",
    "description": "온라인 팀 미팅"
  }
}
```

### 2. 일정 조회 (Action: `LIST`)
- **조건**: 사용자가 일정을 확인 또는 검색하고자 할 때.
- **인자**:
  - `time_min`: 조회 시작 기준 시간 (ISO 8601 포맷, 지정되지 않은 경우 현재 시간)
  - `time_max`: 조회 종료 제한 시간 (ISO 8601 포맷, 지정되지 않은 경우 7일 뒤)
  - `query`: 일정 검색 필터링용 키워드 (선택 사항)

```json
{
  "action": "LIST",
  "arguments": {
    "time_min": "2026-05-26T09:00:00",
    "time_max": "2026-06-02T23:59:59",
    "query": "회의"
  }
}
```

### 3. 일정 삭제 (Action: `DELETE`)
- **조건**: 기존 일정을 제거하고자 할 때.
- **인자**:
  - `query`: 삭제할 일정을 찾기 위한 검색 키워드 또는 고유 일정명 (필수)
  - `target_date`: 삭제 대상 일정이 위치한 특정 날짜 (선택 사항, 모호성 해결용)

```json
{
  "action": "DELETE",
  "arguments": {
    "query": "마케팅 전략 회의",
    "target_date": "2026-05-27"
  }
}
```

### 4. 일치하지 않는 일반 대화 (Action: `CHAT`)
- **조건**: 사용자가 캘린더 작업과 관계없는 일상적인 인사나 기타 대화를 할 때.
- **인자**:
  - `response_text`: 에이전트가 사용자에게 전할 부드러운 일상 대답 메시지 (LLM이 자율적으로 생성)

```json
{
  "action": "CHAT",
  "arguments": {
    "response_text": "안녕하세요! 저는 구글 캘린더 일정을 관리해 드리는 AI 비서입니다. 일정 등록이나 조회를 요청해 보세요!"
  }
}
```

---

## ⚠️ 예외 및 예방 조치 (Edge Cases)

1. **시간 정보의 모호함**:
   - 사용자가 "3시에 미팅 잡아줘"라고만 입력하여 오전/오후 구분이 불명확할 경우, 현재 일시(`current_time`)를 기준으로 미래에 가장 가까운 시간(예: 현재가 오전 11시면 당일 오후 3시, 현재가 오후 5시면 다음 날 오후 3시 혹은 오전 3시 중 문맥상 알맞은 시각)으로 타임스탬프를 보정합니다.
2. **필수 입력값 결여**:
   - 일정을 등록하려는데 제목(`summary`)이 명확하지 않은 경우, "새로운 일정" 또는 "AI 자동 생성 일정" 등의 기본값을 부여하되, 상세 묘사(`description`)에 분석 원문을 함께 기록합니다.
3. **삭제 요청 시 다수의 일정 매칭**:
   - 삭제를 요구했으나 해당 키워드에 매칭되는 일정이 2개 이상일 경우, 삭제를 지시하기 전 사용자에게 삭제할 일정의 날짜나 정확한 시간을 확인하도록 목록을 먼저 조회(`LIST`)하여 제시하는 흐름으로 전환합니다.
