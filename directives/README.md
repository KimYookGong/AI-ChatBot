# Directives Layer 가이드라인 (SOP 작성 가이드)

Directives Layer는 인간의 의도와 행동 방침을 AI 에이전트에게 전달하는 **표준 운영 절차(SOP, Standard Operating Procedure)** 마크다운 문서들의 저장소입니다.

LLM은 확률적으로 작동하기 때문에, 복잡한 비즈니스 로직을 처음부터 끝까지 혼자 추론하게 하면 오류가 누적됩니다. 이를 막기 위해 우리는 명확한 지침서(Directive)를 작성하고, 에이전트가 이 지침에 정의된 개별 도구(Execution 스크립트)를 순차적으로 호출하도록 통제합니다.

---

## 📝 Directive 작성 필수 템플릿

새로운 지침서를 작성할 때는 항상 아래의 구조를 준수해 주세요.

```markdown
# [지침서 제목 - 예: 웹 데이터 수집 및 이메일 보고]

이 지침서가 무엇을 해결하기 위해 존재하는지 목적을 간략하게 설명합니다.

## 🎯 최종 목표 (Objective)
- 에이전트가 궁극적으로 도달해야 하는 상태를 명확히 작성합니다. (예: 주간 판매량 데이터를 추출하여 구글 시트에 기입하고 팀장에게 링크 전달)

## 📥 입력값 및 요구 정보 (Inputs)
- 실행 시 에이전트가 준비하거나 사용자에게 질문해야 하는 필수 정보 목록입니다.
  - 예: `target_url`, `sheet_id`

## ⚙️ 실행 단계 및 사용 도구 (Execution Steps)
에이전트가 순차적으로 실행해야 할 도구(python 스크립트)와 인자(arguments)를 구체적으로 기술합니다.

1. **[1단계] 중간 데이터 준비**
   - **사용 도구**: `execution/sample_tool.py`
   - **동작**: 대상 데이터를 읽어 가공한 후 `.tmp/` 디렉토리에 중간 JSON 파일로 저장합니다.
   
2. **[2단계] 최종 결과 보고**
   - **사용 도구**: (예시) `execution/google_sheet_tool.py`
   - **동작**: `.tmp/`에 저장된 가공 데이터를 Google Sheets API를 사용하여 타겟 시트에 씁니다.

## ⚠️ 예외 상황 및 대처 방안 (Edge Cases & Errors)
- API 호출 제한(Rate Limit), 인증 오류, 잘못된 입력 등의 상황에 에이전트가 어떻게 행동해야 하는지 정의합니다.
  - 예: API 429 오류가 발생하면 5초간 대기 후 최대 3회 재시도할 것. 계속 실패할 경우 사용자에게 알릴 것.
```

---

## 💡 에이전트 주의사항
1. **임시 파일 보존**: 모든 수집 정보 및 파싱 데이터는 항상 `.tmp/` 디렉토리에 고유 파일명으로 임시 보존해야 합니다.
2. **자가 개선(Self-annealing)**: 실행 스크립트(Execution Layer)가 오류로 멈췄을 때, 에이전트는 오류 코드를 해석하여 스크립트의 에러를 수정(fix)하고 다시 테스트한 뒤, 필요하다면 이 Directive의 예외 상황 가이드라인을 보강(update)해야 합니다.
